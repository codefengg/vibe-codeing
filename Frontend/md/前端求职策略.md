# 前端求职策略：给新 Agent 的上下文

这份文档用于让新的 agent 快速理解当前判断框架。

## 用户背景

用户是 6 年 Web 前端，主要写 React hooks + CSS，涉及 Web 浏览器和小程序。当前薪资约 16k，目标 20k+。用户不想要泛泛的“前端知识大全”，而是想要真实求职和面试导向的优先级。

## 核心判断

当前前端找工作最重要的不是会多少零散知识点，而是能不能用项目证明自己不是单纯写页面的人。

最重要的信号是：

1. 真实复杂业务经历
2. React + TypeScript 复杂业务架构能力
3. 性能优化与线上排障能力
4. 工程化和质量保障意识
5. AI 开发工作流能力
6. AI 应用项目差异化

不要把闭包、作用域、原型链这类孤立八股作为主线。它们只在解释真实问题时补充，比如 stale state、竞态、内存泄漏、长任务卡顿。

## 推荐路线

第一阶段：项目包装

把用户过去的业务项目整理成“背景、复杂点、技术方案、取舍、结果、复盘”。至少准备 2 个能讲 10 分钟的项目故事。

第二阶段：React + TypeScript 架构

重点是组件边界、状态归属、服务端状态 vs 客户端状态、复杂表单、复杂表格、权限、hooks 抽象、类型建模。

第三阶段：性能与排障

重点准备首屏慢、白屏、卡顿、包体积、接口慢、React 重渲染、缓存问题、线上回滚。回答时强调排查路径，不要只说懒加载和压缩。

第四阶段：工程化

重点是 Vite、构建优化、代码分割、lint、typecheck、CI、测试、发布、监控、回滚。工程化要讲成“如何保障稳定交付”。

第五阶段：AI 工作流

AI 不是单独热点，而是工程化提效能力。重点讲：AI 读代码、拆需求、出计划、小步实现、生成测试、code review、验证结果、控制无关修改。

第六阶段：AI 应用项目

建议做一个 AI 中后台 Copilot 或类似项目，体现流式 UI、tool calling、结构化输出、权限、错误重试、上下文管理、成本控制。

## 渡一式理解

如果用户提到“渡一前端教育”，需要吸收的不是课程目录，而是学习观：

不是从零重学基础，而是做基础收官。

基础要能解释真实现象：

- 页面为什么白屏
- 页面为什么卡顿
- 状态为什么旧
- 请求为什么竞态
- 缓存为什么出错
- 跨域和登录态为什么异常
- 工程化为什么存在

学习方式是螺旋式的：基础在项目、架构、性能、工程化中不断回到现场。

## 对话风格

用户明确说过：平时对话不要强行用表格。表格主要用于做笔记。正常答复应以自然段为主，简洁、现实、直接。

## 最短答案模板

当前前端找工作最重要的是项目证明能力。你要先把两个真实项目整理成能讲清楚的技术故事，然后围绕 React/TS 架构、性能排障、工程化和 AI 工作流补能力。不要继续扩大知识范围，也不要从零刷基础课。基础只在解释真实问题时补。
